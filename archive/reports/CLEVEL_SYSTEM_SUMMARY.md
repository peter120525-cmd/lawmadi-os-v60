## 🛠️ 기술 구현

### CLevelHandler 클래스

**파일**: `agents/clevel_handler.py`

**주요 메서드:**

1. **detect_clevel_call()**
   - Query에서 C-Level 임원 호출 감지
   - 이름 또는 전문 영역 키워드 매칭

2. **get_clevel_system_instruction()**
   - C-Level 임원별 맞춤형 시스템 지시 생성
   - 전문 분야 관점 명시

3. **should_invoke_clevel()**
   - C-Level 호출 여부 및 모드 결정
   - direct: 직접 호출
   - swarm: Swarm에 포함
   - none: 일반 처리

### main.py 통합

```python
# Startup
clevel_handler = CLevelHandler(core_registry)

# /ask 엔드포인트
clevel_decision = clevel_handler.should_invoke_clevel(query)

if clevel_decision.get("mode") == "direct":
    # C-Level 직접 호출
    exec_id = clevel_decision.get("executive_id")
    system_instruction = clevel_handler.get_clevel_system_instruction(
        exec_id, SYSTEM_INSTRUCTION_BASE
    )
    # ... C-Level 전용 응답 생성
```

## 💡 각 임원의 관점 차이

### 서연 (CSO) - 전략적 관점

**분석 방식:**
1. 전략적 우선순위 설정
   - 핵심 쟁점은?
   - 먼저 해결할 것은?
   - 장기 vs 단기 관점

2. 의사결정 지원
   - 선택지와 장단점
   - 리스크와 기회 분석
   - 최선의 전략 제안

3. 로드맵 제시
   - 단계별 실행 계획
   - 예상 장애물과 대응
   - 성공 지표(KPI)

**응답 형식:**
```markdown
[서연 (CSO) 전략 분석]

1. 요약
   → 전략적 핵심 판단

2. 법률 근거
   → 법적 근거 + 전략적 해석

3. 시간축 분석
   → 전략적 타임라인

4. 절차 안내
   → 전략적 실행 계획

5. 참고 정보
   → 전략적 인사이트
```

### 지유 (CTO) - 기술적 관점

**분석 방식:**
1. 시스템 아키텍처 분석
   - 법률 시스템의 구조
   - 법률/규정 간 의존 관계
   - 무결성 검증

2. AI 검증 및 무결성
   - 논리적 정합성
   - 법률 해석 일관성
   - 근거 타당성 검증

3. 성능 최적화
   - 효율적인 절차
   - 불필요한 단계 제거
   - 시간/비용 최소화

4. 기술 가이드
   - 법률 테크 활용
   - 디지털 증거 관리
   - 온라인 절차 활용

**응답 예시:**
- "데이터 레이어에서 SSOT 원칙 적용"
- "종단간 암호화로 무결성 보장"
- "결정론적 추론 엔진 설계"

### 유나 (CCO) - 콘텐츠 관점

**분석 방식:**
1. 사용자 경험 최적화
   - 일반인이 이해하기 쉬운 설명
   - 법률 용어의 친절한 풀이
   - 시각적 구조화

2. 콘텐츠 설계
   - 정보의 계층 구조
   - 핵심 메시지 우선 제시
   - 세부 사항은 참고로

3. 인터페이스 디자인
   - 단계별 명확한 가이드
   - 체크리스트 형식
   - 시각적 요소 활용

4. 커뮤니케이션 전략
   - 감정적 공감
   - 불안 해소
   - 실행 가능한 조언

**응답 특징:**
- 표와 타임라인 적극 활용
- 단계별 체크리스트
- 이모지로 가독성 향상
