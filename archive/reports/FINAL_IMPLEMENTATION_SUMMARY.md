## 🎯 사용 예시

### 예시 1: 복합 법률 사안 (Swarm 자동 활성화)

**입력:**
```
아버지께서 제 명의로 아파트를 구입하셨습니다. (명의신탁)
그런데 제가 그 아파트를 임대를 주었고, (임대차)
지금 제 빚 때문에 경매가 진행 중입니다. (경매)
```

**시스템 처리:**
```
1. 도메인 탐지: 명의신탁, 임대차, 경매
2. Leader 선택: 세움(L57), 온유(L08), 결휘(L10)
3. Swarm 모드 활성화
4. 3명 병렬 분석
5. 마디가 통합
```

**응답:**
```markdown
[마디 통합 리더 답변]

### 2. 📚 법률 근거

| 전문 분야 | 법률/판례 | 핵심 내용 |
|-----------|----------|----------|
| **휘율 (민사법)** | 부동산실명법 제4조 | 명의신탁 무효 |
| **결휘 (민사집행)** | 강제집행 유효성 | 경매 진행 가능 |
| **온유 (임대차)** | 주택임대차보호법 | 우선변제권 |
```

### 예시 2: C-Level 직접 호출

**입력:**

```
지유님, 법률 시스템 아키텍처를 무결성 있게 설계하는 방법은?
```

```
1. C-Level 탐지: "지유" → CTO
2. 직접 호출 모드
3. CTO 전용 시스템 지시 적용
```

```markdown
[지유 (CTO) 기술 분석]

### 1. 요약
시스템 무결성은 세 가지 레이어로 보장:
1. 데이터 레이어: SSOT 원칙
2. 프로세스 레이어: 종단간 암호화
3. 추론 레이어: 결정론적 추론
```
