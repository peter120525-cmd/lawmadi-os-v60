## 1️⃣ 시스템 아키텍처

### Before: 단일 Leader 선택

```python
def select_swarm_leader(query, leaders):
    # 키워드 매칭으로 1명만 선택
    if "전세" in query or "임대" in query:
        return leaders["L08"]  # 온유 (임대차)
    ...
    return leaders["L60"]  # 마디 (기본)

# 결과: 단일 Leader의 관점만 반영
```

**문제점:**
- ❌ 복합 사안에서 일부 법률 영역 누락
- ❌ 단일 전문가 관점의 한계
- ❌ 교차 법률 관계 파악 부족

### After: Swarm Orchestrator

```python
class SwarmOrchestrator:
    def orchestrate(self, query):
        # 1. 도메인 탐지
        domains = self.detect_domains(query)
        # → 발견: ["명의신탁", "임대차", "경매"]

        # 2. Leader 선택
        leaders = self.select_leaders(domains)
        # → 선택: [L57-세움, L08-온유, L10-결휘]

        # 3. 병렬 분석
        results = self.parallel_swarm_analysis(leaders)

        # 4. 결과 통합
        final = self.synthesize_swarm_results(results)
        return final
```

**해결:**
- ✅ 복합 사안의 모든 법률 영역 커버
- ✅ 다중 전문가 관점 반영
- ✅ 교차 법률 관계 종합 분석
